# Release v2.7.0 - 核心安全分析与深度扫描增强

## 主要更新

### 🚀 新功能与重大改进

#### 1. 新增 HTML 交互式安全报告
- 重构了安全审查模块的终端输出方式，现已支持 **HTML 交互式报告** 生成。
- 报告内置漏洞分布饼状图、响应式搜索和选项卡分类过滤，且完全采用**单文件架构**（无需外挂 JS/CSS），分发与审查变得非常直观。
- 命令行支持：所有附带敏感信息扫描的命令将会在工作目录下同步生成详细报告。

#### 2. 新增 `scan-only` 独立安全审查模式
- 解耦了代码解密、反编译与敏感信息提取流程。
- 加入全新的 `scan-only` 子命令，专门用于对已导出或别人发送的源码文件夹执行深层检查，速度更快更直接。

#### 3. 改进的批处理调度 `all` 支持指定 `-id`
- `-all` 指令全面进化，不仅支持一键挂载微信存储引擎下全部缓存数据。
- 新增通过 `-id="wx123,wx234"` 直接使用逗号批量提供需解析的应用清单，不再依赖全局重爆破。
- 新增 `-id-file=target.txt` 批量行读取模式，支持多环境无缝作业联动。

#### 4. [黑科技] 极速提取 Windows PC 端加密主包真实名称
- 解决了新规下用户沙盒（如：`radium/users/.../applet/`）内无法单纯凭借路径下的 `appconfig.json` 定位应用真实名称的痛点。
- 突破了在不对源包解密落盘的情况下，通过热读缓冲流直达 `app-config.json` 获取 `navigationBarTitleText` / `appName` 字段；
- **纯 Golang 静态实现，不依赖 CGO 环境 / SQLite 黑盒驱动**，解析速度稳定在 < 10ms，使用 `scan` 时终端列表自动输出友好的中文应用名。
- 注：第三方服务依赖（PluginCode 包）或微信小游戏等内部结构无固定 JSON 的包体依然不作命名展示，此为正常现象。

#### 5. GitHub Actions 全链路自动化支持
- 新增对远端仓库的事件挂钩。
- PR 到 Main 时能自动通过 Lint 和运行环境编译校验。
- 新的 `v*.*.*` Release 标签能够由远端 Worker 并发编译 Linux, MacOS（Intel/M系列）, Windows 全端包供用户下载。

## 推荐用法

```bash
# 1. 精确获取所有已缓存（包含隔离环境）的微信小程序列表，带应用名称
gwxapkg scan

# 2. 定点执行批量渗透扫描并一键生成 HTML 安全测试报告
gwxapkg all -id="wxa8da525af05281f3,wx8568bdcce6406770" -sensitive=true

# 3. 单纯复审历史解包文件，直接查阅结果
gwxapkg scan-only -in=./history_pkg_source/ -out=./reports/
```
